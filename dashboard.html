Agent Quick Start

Herd is a reverse proxy between you and Ollama backends. Send requests to Herd — it picks the best backend based on model availability, GPU load, and tags.

1. Discover Models

GET /v1/models

Returns all models available across healthy backends. Always check before requesting an unknown model.

2. Send Chat Requests

POST /v1/chat/completions
Content-Type: application/json

{
  "model": "qwen2.5-coder:32b",
  "messages": [
    {"role": "user", "content": "Hello"}
  ],
  "stream": true
}

OpenAI-compatible. Your existing client libraries work unchanged. Always specify the model for optimal routing.

3. Target Specific Backends

X-Herd-Tags: gpu,fast

Add this header to route only to backends matching all specified tags. Use for workload isolation.

4. Trace Requests

X-Request-Id: agent-task-42

Send a correlation ID to trace requests in logs. If omitted, Herd generates a UUID v4 and returns it.

Routing Strategies

The active strategy is shown in the header badge. It determines how Herd picks a backend for your request.

StrategyBehavior
model_awarePrefers backends with your model already loaded. Avoids cold starts. (default)
priorityAlways routes to the highest-priority healthy backend.
least_busyRoutes to the lowest GPU utilization.
weighted_round_robinDistributes across backends weighted by priority.

Best Practices

Do

  • Always specify "model" in requests
  • Use "stream": true for long responses
  • Query /v1/models before requesting unknown models
  • Send X-Request-Id for traceability
  • Use X-Herd-Tags for workload isolation
  • Retry on 503 — circuit breaker will recover backends

Don't

  • Don't hard-code backend URLs — always go through Herd
  • Don't omit the model — routing can't optimize without it
  • Don't ignore 429 — back off when rate-limited
  • Don't send huge payloads — 10 MB body limit
